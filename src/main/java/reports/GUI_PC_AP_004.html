10.0Type the description of the projecttypeCem Kaner defines testing as a technical investigation of a product under test in order to provide information regarding the quality of the software to the different actors involved in a project. Decisions can be made from the information obtained from testing. Decisions can range from when to release a product into production, knowing the risks that this implies, to how to improve the different areas within the company. In short, testing is an agent of change, the important thing is to interpret the information obtained so that all actors can act in a timely manner where necessary. In software, trust is an important element since certain failures can have undesirable consequences such as loss of money, business and even lives, depending on how critical the domain in which the software interacts is. The tests give added value to each project, giving confidence to the different actors.
